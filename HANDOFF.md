# 작업 인수인계 메모 (다른 컴퓨터에서 이어가기용)

- 작성일: 2026-06-21
- 목적: 다른 컴퓨터에서 작업을 이어가기 위해, 지금까지의 상황과 **결정 대기 중인 지점**을 정리.
- 읽는 순서: 2번(지금 멈춘 곳)이 핵심. 1번은 배경, 3~5번은 실행 정보.

---

## 1. 프로젝트 전체 상태 (요약 — 배경)

사이트(lowstamp.com)는 **라이브**이고 개발은 사실상 끝났다. 6개 언어, 개별 게임 페이지(`/game/:appid`), SEO/사이트맵, 쿠키 동의(Google 동의 모드 v2)까지 다 됨.

**애드센스 상태**: 애드센스 "사이트" 메뉴에 `lowstamp.com`(www 없이) 등록 완료. 현재 화면에 "준비 중" + ads.txt "찾을 수 없음" 표시.
- 확인 결과 **ads.txt는 실제로 정상 서빙 중**임. `https://lowstamp.com/ads.txt` → HTTP 200, `text/plain`, 내용 정확(`google.com, pub-6033148215263757, DIRECT, f08c47fec0942fa0`). 구글 광고 로봇(AdsBot-Google) UA로 요청해도 정상 응답. robots.txt도 전체 허용.
- 즉 **고칠 것 없음**. 구글이 아직 재크롤 안 한 "시간 문제"라 며칠~2주 대기. 신규 사이트는 흔한 상황.
- 승인되면 할 일: `web/src/lib/ads.js`의 빈 `SLOTS`(gameTop, guideMid, dealsInline, sidebar, homeTop)에 애드센스 대시보드에서 만든 슬롯 번호를 넣으면 광고가 켜짐.

**자동화**: 매일 한국시간 아침 9시(cron `0 0 * * *`, GitHub Actions) 크롤링 → D1 저장 → 게임 페이지/사이트맵 자동 생성. **DB는 누적**(옛 게임 안 지움 — 크롤러에 DELETE 없음). 검색(`/api/search`)은 전체 `games` 테이블 대상이라 옛 게임도 검색됨. 유지보수로 가끔 GitHub Actions 초록불 확인 + 1년마다 도메인 연장만 챙기면 됨. (저장소 2개월 무활동 시 cron 자동 중지 주의.)

## 2. 지금 진행 중인 작업 (결정 대기 중) ← **여기서 멈춤**

**목표**: 홈의 "오늘의 최저가" 탭과 "할인 중" 탭 리스트를 **"더 보기" / 무한 스크롤**로 더 많이 볼 수 있게 한다. (사용자 의도: 리스트가 인위적으로 제한된 느낌을 없애고, 누적되는 가격 추적기답게 많이 보이게.)

**핵심 발견 (중요)**:
- 워커가 응답 개수를 하드캡 함: **할인 중(`/api/deals`) = 최대 120개**, **오늘의 최저가(`/api/lowest-today`) = 40개 고정**. 라이브에서 `limit=500`을 줘도 120개만 돌아옴 → 캡이 실제로 걸린다(둘 다 그 이상 게임이 있음).
- 워커에 **offset(페이지네이션) 파라미터 없음**. 그래서 프런트만으로는 120/40을 넘길 수 없음.
- 워커는 **깃 자동배포가 안 됨**: 저장소에 `wrangler.toml` 없음, 워커 배포용 워크플로 없음(`.github/workflows/`엔 `crawl.yml`만). → 워커를 고치면 **클라우드플레어 대시보드 > Workers > gamgap-api**에 `worker.js`를 붙여넣고 **Deploy를 수동으로** 눌러야 반영됨.

**사용자에게 던진 질문 (아직 최종 선택 안 됨 — 이걸 정하면 바로 구현)**:
- **옵션 A "무제한"**: `worker.js`에 offset 페이지네이션 추가 + 캡 상향/제거 → 프런트에서 무한 스크롤로 끝까지. 단 **워커 수동 배포 필요**. (프로젝트 규칙 "백엔드 건드리지 말 것"과 충돌하지만, 이번 변경은 읽기 응답을 더 주게만 하는 **안전한 추가**라 가격 판정/데이터엔 영향 없음. 사용자 승인 전제.)
- **옵션 B "프런트만"**: 워커 안 건드리고 화면 쪽만 점진적 표시("더 보기" 버튼/무한 스크롤). 배포 불필요, 바로 적용. 단 **할인 중 120개·최저가 40개 상한 유지**.

> 참고: 사용자가 옵션 선택 대신 "이 인수인계 md를 만들어 달라"고 요청해서 이 파일을 작성함. **다른 컴퓨터에서 이어갈 때, 먼저 사용자에게 A/B 중 무엇으로 할지 확인하고 진행할 것.**

## 3. 관련 파일·위치 (실행 정보)

- **프런트 데이터 호출**: `web/src/api.js`
  - `getDeals(limit = 60, cc)` → `/api/deals?limit=...`
  - `getLowestToday(cc)` → `/api/lowest-today`
  - (옵션 A면 여기에 `offset` 인자 추가)
- **프런트 상태/렌더**: `web/src/App.jsx`
  - 로더: `loadDeals`(약 195~204줄, 현재 `getDeals(120, cc)`), `loadLowest`(약 184~193줄)
  - 렌더: `Section`(약 48~77줄 — 최저가/검색 공용), `DealsView`(약 112~166줄 — 할인 중, **필터 적용 후 `filtered` 렌더**)
  - 여기에 "표시 개수" 상태 + "더 보기" 버튼 또는 IntersectionObserver(무한 스크롤) 추가.
- **정렬·필터**: `web/src/lib/dealSort.js` — `applyDealOpts(rows, opts)`. 할인 중 탭은 필터/정렬 후 렌더하므로, "더 보기"는 **필터된 결과(filtered)** 기준으로 잘라 보여줘야 함.
- **백엔드(옵션 A만)**: `worker.js`
  - `/api/lowest-today` → `LIMIT 40` (약 115~127줄), region 버전 포함
  - `/api/deals` → `const limit = Math.min(Number(...)||60, 120)` (약 129~141줄)
  - offset 받게 수정 + 캡 상향. region 쿼리도 동일하게.

## 4. 구현 스케치

**옵션 B (프런트만, 빠른 길)**:
- `App.jsx`에 `visibleCount` 상태(예: 초기 24). `Section`/`DealsView`에서 `rows.slice(0, visibleCount)`만 렌더. 목록 맨 아래 "더 보기" 버튼 → `visibleCount += 24`. (또는 sentinel div + IntersectionObserver로 보이면 자동 증가.)
- deals는 이미 120개 다 받아오므로 추가 네트워크 호출 없음. lowest는 40개. 탭 전환/필터 변경 시 `visibleCount` 리셋.
- 광고 인라인 위치(`i === 5`의 `dealsInline`)와 충돌 안 나게 슬라이스 후 인덱스 기준 유지.

**옵션 A (무제한)**:
- `worker.js`: `/api/deals`, `/api/lowest-today`에 `&offset=` 받아 `... LIMIT ? OFFSET ?` 추가, 페이지당 개수 정함(예: 60). region 쿼리도 동일.
- `api.js`: `getDeals`/`getLowestToday`에 `offset` 인자 추가.
- `App.jsx`: 스크롤 끝 도달 시 다음 페이지 fetch해서 기존 rows에 **append**(누적). 무한 스크롤 sentinel.
- **배포**: 클라우드플레어 대시보드에서 gamgap-api 워커에 `worker.js` 붙여넣고 Deploy. (또는 `wrangler.toml` 만들어 `wrangler deploy`.) — 이건 사용자가 수동으로.

## 5. 곁다리: "한글↔영문 검색 별칭"이란? (사용자 질문에 답한 내용)

화면 표기를 언어별로 바꾸는 게 **아님**(그건 이미 됨). **검색 매칭**을 돕는 기능. 예: 사용자가 "위처3"이라 쳐도 데이터가 "The Witcher 3"이면 지금은 못 찾음. 별칭을 붙이면 "위처3=The Witcher 3", "갓오워=God of War", "젤다=Zelda"처럼 줄임말·다른 언어 표기를 같은 게임으로 이어줘서 어떻게 쳐도 찾게 함. (할 일 목록의 성장 항목 — 별도 작업.)
